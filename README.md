# TrangTuyenSinhLangCS — Students Documents

This project includes a **student downloads page** at `pdf.html` that shows PDF/DOCX files from the `documents/` folder in a sortable grid.

## Why we need `documents/manifest.json`

Browsers can’t automatically list files from your local `documents/` folder. So we maintain an index file:

- `documents/manifest.json` — list of `documents/*.pdf` and `documents/*.docx` used by `pdf.html`

When you add/remove documents, regenerate the manifest.

---

## Option A (Recommended): Regenerate once

1. Copy your new files into the `documents/` folder (same folder as `manifest.json`).
2. Regenerate the manifest by running:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\generate-documents-manifest.ps1
```

3. Refresh `pdf.html` in your browser.

---

## Option B: Fully automatic (watcher)

1. Copy your new files into `documents/`.
2. Start the watcher (keep this PowerShell window open):

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\watch-documents-manifest.ps1
```

The watcher will automatically update `documents/manifest.json` whenever you add/change/rename `.pdf` or `.docx` files.

`pdf.html` also polls the manifest periodically, so the grid updates without needing manual edits.

---

## Note about opening `pdf.html`

For the manifest fetch to work reliably, open `pdf.html` via a local server when possible (not `file://...`).
If you open directly via `file://`, some browsers may block `fetch()` of `./documents/manifest.json`.

---

## Where to edit

You usually won’t edit these files directly:

- `pdf.html`
- `documents-grid.js`
- `documents/manifest.json` (generated by the scripts)

Only use the PowerShell scripts to update the manifest.
